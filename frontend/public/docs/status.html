01 What the twin is

A digital replica of the real plant's physics — not a recording of it. Feed it the operator's decisions (load, setpoints, which units run) and it computes every consequence: power, temperatures, flows, and efficiency. Operators work it through a 2D SCADA P&ID or a fully modelled interactive 3D plant room.

  • Exact where physics is exact. Energy balances, heat rejection and the COP identity carry zero fitted parameters — they cannot drift.
  • Fitted where the plant is empirical. Only the chiller part-load curve and condenser-lift sensitivity are learned from data.
  • Honest about its limits. Every output is flagged calibrated or extrapolated against the region the data actually covers.

02 How it was calibrated

A grey-box approach: keep the physics structure fixed and exact, and fit the smallest possible set of coefficients from the BMS trend. The chiller curve is an affine (fixed-loss + per-load) fit; the condenser sensitivity was extracted from the plant's natural weather variation across the month.

  • Not black-box ML. A linear regression — chosen because the physics is near-linear over the operating band and stays interpretable and safe to extrapolate.
  • Load-dependent efficiency. The affine curve reproduces the real plant getting more efficient as load rises — a proportional model could not.
  • One known gap. CHWS sensitivity can't be learned from this data (the setpoint never moved); it needs a step test.

04 What's built

  • 2D SCADA view. BMS-style P&ID with faceplates, editable setpoints, duty rotation, before→after impact, plant summary.
  • 3D plant view. Full three.js plant room — reference-modelled centrifugal chillers, two-cell crossflow towers, end-suction pump sets, tanks and valves — live data tiles on every unit, orbit/zoom, click-to-select.
  • Calibration guard. Live badge flags whether current inputs sit inside the data-validated region or are being extrapolated.
  • Scoring API. npm run api exposes stateless, deterministic endpoints: POST control inputs → efficiency + full state + confidence flag. Batch mode for ML sweeps.
  • Validation suite. Replay scripts (validateRow86, validateMvWindow) rerun the full evidence chain on demand — a regression gate for any model change.
  • ML-ready. Every response carries the calibration status so an optimiser can't chase savings outside the validated envelope.

⚠ Honest limits — state these upfront

  • Can't beat the noise floor. At the current operating point the twin is as accurate as the data allows (±0.29%); the plant itself doesn't repeat exactly.
  • Narrow calibrated band. Confident within RT ≈ 2600–3470, CHWS ≈ 7.5, CWS ≈ 29. Outside, outputs are physics extrapolations, flagged as such.
  • CHWS sensitivity unproven. The setpoint never moved in the data — a step test is required before trusting CHWS-reset recommendations in closed loop.
